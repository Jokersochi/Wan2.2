## 2024-05-24 - Insecure Deserialization in PyTorch Model Loading
**Vulnerability:** The codebase was loading PyTorch model weights using `torch.load()` without the `weights_only=True` flag. Since `torch.load()` uses Python's `pickle` module by default, this creates an insecure deserialization vulnerability that could allow arbitrary code execution if a malicious checkpoint file is loaded.
**Learning:** PyTorch defaults to full pickle deserialization unless explicitly restricted. Even when loading checkpoints from local paths or trusted sources, the defense-in-depth principle requires restricting deserialization to expected data structures to prevent exploitation if a path is compromised or manipulated.
**Prevention:** Always set `weights_only=True` when using `torch.load()` to load model weights or state dictionaries. This restricts the unpickler to only load tensors, primitive types, and dictionaries, preventing the execution of arbitrary code embedded in the pickle file.
