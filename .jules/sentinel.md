## 2026-04-10 - [Insecure Deserialization in PyTorch]\n**Vulnerability:** The `torch.load()` function was used without `weights_only=True`, exposing the application to insecure deserialization vulnerabilities via malicious pickle files.\n**Learning:** By default, `torch.load()` uses Python's `pickle` module, which can execute arbitrary code during unpickling. Because state dictionaries should only contain tensors and basic Python types, restricting the unpickler is a highly effective security measure.\n**Prevention:** Always use `torch.load(..., weights_only=True)` when loading weights, state dictionaries, or arbitrary PyTorch checkpoints unless explicit, trusted execution of embedded code is fundamentally required.
